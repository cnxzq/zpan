# 使用指南

## 启动服务

### 基本启动

```bash
# 使用默认配置（端口 8090，当前目录，用户名 admin，密码 admin123）
ark-pan
```

### 自定义端口和目录

```bash
ark-pan 8080 /home/user/myfiles
```

### 使用 .env 配置文件启动

```bash
# 在项目目录下
npm start

# 或者全局安装后
ark-pan
```

## 访问服务

启动后，在浏览器打开：

```
http://127.0.0.1:8090
```

会弹出认证对话框，输入配置的用户名和密码即可访问。

## 功能说明

### 目录浏览

进入目录后，你会看到：

- 当前目录下的所有文件和子文件夹列表
- 文件图标（区分不同文件类型）
- 文件大小
- 最后修改时间
- **📤 上传文件** 按钮

点击文件名可以直接下载文件，点击文件夹名称可以进入子目录。

### 上传文件

1. 在目录页面点击 **📤 上传文件** 按钮
2. 选择要上传的一个或多个文件
3. 点击 **开始上传** 按钮
4. 等待上传完成后会自动跳回目录页面

文件会上传到当前浏览的目录下。

### 下载文件

点击文件列表中的文件名，浏览器会自动下载该文件。

## 使用示例

### 示例 1：分享当前目录

```bash
ark-pan
```

访问 `http://127.0.0.1:8090` 即可浏览当前目录的文件。

### 示例 2：分享特定目录到 8080 端口

```bash
ark-pan 8080 /home/user/documents
```

### 示例 3：允许局域网访问

修改 `.env`：

```env
HOST=0.0.0.0
PORT=8090
```

启动后，局域网内其他设备可以通过你的 IP 访问，例如：

```
http://192.168.1.100:8090
```

### 示例 4：自定义认证信息

```bash
# 使用环境变量
USERNAME=alice PASSWORD=my-secret ark-pan 8080 ~/share
```

或者在 `.env` 中：

```env
USERNAME=alice
PASSWORD=my-secret
PORT=8080
STATIC_ROOT=/home/alice/share
```

## 注意事项

### 安全性

- Ark Pan 只提供基础的 Basic Auth 认证，不要用于存放极度敏感的文件
- 建议使用强密码
- 如果暴露到公网，建议配合 HTTPS 使用（可以通过 Nginx 反向代理实现）

### 文件大小限制

- 默认最大支持 10GB 单文件上传
- 可以通过 `MAX_FILE_SIZE` 环境变量调整限制
- 实际可用空间受限于服务器磁盘空间

### 符号链接

- Express 静态文件服务默认跟随符号链接
- 确保 `STATIC_ROOT` 目录下的符号链接指向的文件是你希望分享的

## 故障排除

### 无法访问服务

- 检查端口是否被占用：`lsof -i :8090`
- 检查防火墙是否允许访问
- 如果绑定 `127.0.0.1`，只能本机访问，需要改为 `0.0.0.0` 允许外部访问

### 上传失败

- 检查文件大小是否超过 `MAX_FILE_SIZE` 限制
- 检查目标目录是否有写入权限
- 检查磁盘空间是否充足

### 认证不通过

- 检查用户名和密码是否正确
- 检查环境变量或 `.env` 配置是否正确
- 清除浏览器缓存后重试
